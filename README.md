# Projeto de Testes Manuais de API - ServeRest

Este projeto faz parte do meu portfólio de QA Manual. Ele contém a especificação e o planejamento de testes para a API pública ServeRest, cobrindo cenários de autenticação, gerenciamento de produtos e regras de negócio de carrinho de compras.

## 🛠️ Ferramentas Utilizadas
* **Postman** (Para execução dos testes manuais)
* **ServeRest API** (Ambiente de testes)

## 📋 Cenários de Teste Cobertos
* **TC 001 a TC 004:** Testes de Autenticação (Fluxos positivos, negativos e validação de campos obrigatórios).
* **TC 005 e TC 006:** Testes de Cadastro de Produtos (Fluxo positivo e validação de duplicidade de registro).
* **TC 007 e TC 008:** Testes de Carrinho de Compras (Fluxo positivo e validação de regra de negócio de estoque).

## 🚀 Como Executar os Testes
1. Abra o arquivo `casos_de_teste.md` para visualizar o roteiro passo a passo.
2. (Opcional) Importe o arquivo `serverest-collection.json` no seu Postman para visualizar as requisições prontas.
